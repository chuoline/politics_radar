# Decision Log — PoliticsRadar

## 記録日
2025-XX-XX

## 対象
PoliticsRadar  
設計憲章 v1.0 準拠状態における「次フェーズ（拡張）検討」

---

## 目的

本ドキュメントは、PoliticsRadar において  
**「なぜ今、機能追加や拡張を行わない判断をしたか」**を記録するためのものです。

結論ではなく、  
判断に至った条件・懸念・保留点を残すことを目的とします。

---

## 前提

- PoliticsRadar は、政治を評価・断罪・採点するためのツールではない
- 国家として発せられた言葉を、原文と時間・条件の中に置き直すための補助線である
- 評価・結論・判断の代行は行わない
- README の最小公開文が成立し続けることを、設計の最低条件とする

---

## A. 深度（言及条件）拡張の検討

- 深度語彙を増やすことで、上下関係（進捗・成果）に見えるリスクが高い → Yes
- 追加深度が評価語に置き換えられて説明される可能性 → Yes
- 現行4語彙で説明できない事象の存在 → No
- 面ビュー単体で結論が出た気になる危険 → Yes
- 線→点の遷移で十分代替可能 → Yes

**判断**  
現行の4語彙（言及／方向性／制度／実行条件）を維持し、拡張しない。

---

## B. 対象データ拡張の検討（官邸以外）

- 人物・党派評価への引きずられ → Yes
- 比較・優劣を誘発する危険 → Yes
- 説明責任補助としての必然性 → 保留
- 原文回帰導線の強度維持 → No
- 拡張しないことで失われる価値 → 保留

**判断**  
現時点では対象拡張を行わない。

---

## C. UI・可視化の強化検討

- 視覚強化による重要度・成果誤読 → Yes
- 説明不要で分かった気になる危険 → Yes
- 沈黙で閉じる運用への影響 → Yes
- 現行UIの地味さ → 設計選択

**判断**  
UI・可視化の強化は行わない。

---

## D. 分析・数値化の検討

- 数値化によるランキング誤読回避 → No
- 配置のみで目的達成可能 → Yes
- 分析欲求の主体 → 作り手側
- 分析なしで説明責任は成立 → Yes

**判断**  
分析・数値化機能は実装しない。

---

## E. 公開・拡散に関する検討

- 拡散意識による言い切り誘発 → Yes
- 説明省略による評価誤読 → Yes
- 静かな公開で守れる価値 → 文脈・沈黙・誤用耐性
- 反応ゼロでも成立するか → Yes

**判断**  
拡散を目的とした設計変更は行わない。

---

## F. やらないことリスト（再確認）

- 評価しない → 維持
- 採点しない → 維持
- ランキングしない → 維持
- 即答を助けない → 維持
- 結論を代行しない → 維持

---

## 最終確認

> 機能追加後も、README の最小公開文はそのまま成立するか

**回答**：No

---

## 総合結論

- 次フェーズにおいて **実装すべき追加要素は存在しない**
- 現在の PoliticsRadar は **設計的に完成状態**
- 今後進むとすれば、実装ではなく  
  **運用・観察・記録**のフェーズとする
